AI assistants — by design

Why Docker IDEs

Everything the course needs. Nothing it doesn't.

No compiler hunting, no version mismatches, no "it works on my machine." Install Docker Desktop, paste one command, and start writing code.

One-command setup

The only prerequisite is Docker Desktop. A single docker run pulls the image, seeds a starter program, and serves the IDE at a fixed local port.

Real VS Code, in the browser

Powered by code-server: the full VS Code editing experience — integrated terminal, extensions, Run button — with nothing to install or configure locally.

Identical for everyone

Every student — and every grader — gets the same compiler versions, same debugger, same settings. Assignments behave the same on every machine.

Your files are sacred

Work lives in a normal folder on your own computer, never inside the container. Autosave writes to disk as you type — deleting the container can't take your work with it.

No AI, by design

No Copilot, no chat, no agents. Every AI surface in VS Code is deliberately disabled through pinned settings, per the course philosophy. You write the code.

Light on resources

Containers run only when you start them and never auto-restart. On modest hardware, an optional flag caps the IDE's memory and CPU so runaway code can't slow your machine.

The Environments

Pick your course. Both can run side by side.

Each IDE's port is its course number, so the two never conflict.

UNLV C++ IDE

CS 135

A complete C++ toolchain: g++ 11.4, gdb, cmake, valgrind, clang-format, and clangd code intelligence. One-click compile and run with warnings on.

  • Image: seancnc/unlv-cpp-ide
  • Platforms: Intel/AMD + Apple Silicon (native on both)
  • Opens at: http://127.0.0.1:8135

UNLV x86 Assembly IDE

CS 218

An x86-64 Linux assembly toolchain: yasm and nasm assemblers, GNU ld, gdb, and make. The Run button assembles, links, and executes your .asm files.

  • Image: seancnc/unlv-x86-ide
  • Platforms: x86-64 by design (emulated on Apple Silicon)
  • Opens at: http://127.0.0.1:8218

Get Started

Up and running in three steps

Works the same on Windows, macOS, and Linux.

  1. Install Docker Desktop

    Download Docker Desktop, install it, and make sure it's running (the whale icon in your menu bar or system tray).

  2. Run the IDE you need

    C++ IDE (CS 135):

    docker run -it --name unlv-cpp-ide -p 127.0.0.1:8135:8080 -v "$HOME/UNLV/cpp-workspace:/home/coder/workspace" seancnc/unlv-cpp-ide

    x86 Assembly IDE (CS 218):

    docker run -it --platform linux/amd64 --name unlv-x86-ide -p 127.0.0.1:8218:8080 -v "$HOME/UNLV/x86-workspace:/home/coder/workspace" seancnc/unlv-x86-ide

    On Windows, run the same command in PowerShell with the path in Windows form: -v "$HOME\UNLV\cpp-workspace:/home/coder/workspace".

  3. Open your browser

    Go to http://127.0.0.1:8135 (C++) or http://127.0.0.1:8218 (x86). The IDE opens straight into your workspace with a starter program ready to run. Each guide's page covers verifying, stopping, and updating.

Returning students

The old seancnc/unlv-cs-ide image is retired. Switch to seancnc/unlv-cpp-ide (or seancnc/unlv-x86-ide for assembly) using the commands above — the new images work on both Intel/AMD and Apple Silicon machines.
