Quick start

  1. Install Docker Desktop

    Download Docker Desktop and make sure it is running.

  2. Run the IDE

    docker run -it --name unlv-cpp-ide -p 127.0.0.1:8135:8080 -v "$HOME/UNLV/cpp-workspace:/home/coder/workspace" seancnc/unlv-cpp-ide

    On Windows, run the command in PowerShell with -v "$HOME\UNLV\cpp-workspace:/home/coder/workspace". On native Linux (not Docker Desktop), create the folder first with mkdir -p ~/UNLV/cpp-workspace, otherwise Docker creates it owned by root.

  3. Open the IDE and verify

    Go to http://127.0.0.1:8135. The IDE opens into your workspace with hello.cpp seeded. In the IDE's terminal:

    g++ hello.cpp -o hello && ./hello

    You should see Hello, C++!. Clicking the Run button on hello.cpp does the same thing.

Never drop the -v part of the command

The -v flag is what keeps your files in a normal folder on your own computer. Without it, everything you write lives only inside the container and is permanently deleted when the container is removed — which the update steps below do.

What's inside

  • g++ 11.4 and gcc (via build-essential), gdb 12.1, cmake, valgrind, clang-format, clangd
  • Extensions: clangd (code intelligence), Code Runner, Clang-Format
  • The Run button compiles with g++ -std=c++14 -Wall -g and runs in the integrated terminal
  • Autosave is on — your work is written to disk as you type
  • Starter program hello.cpp, seeded only when the workspace folder is empty; existing work is never overwritten
  • No Copilot, no chat, no AI assistance of any kind — deliberately disabled, per the course philosophy

Stopping and restarting

Closing the browser tab does not stop the IDE — the container keeps running and using RAM until you stop it. When you're done, press Ctrl+C in the terminal you started it from, click stop in Docker Desktop, or run:

docker stop unlv-cpp-ide

To start it again later (the container already exists — no need to repeat the long command):

docker start unlv-cpp-ide

Then reopen http://127.0.0.1:8135 — everything reconnects exactly as you left it. Nothing auto-restarts: the run command deliberately has no --restart flag.

On a machine with 8 GB of RAM or less

Cap how much the IDE may use by adding --memory=2g --cpus=2 right after docker run -it — everything else in the command stays the same. An accidental infinite loop in your program then can't slow down the rest of your computer.

Updating

Your files live in ~/UNLV/cpp-workspace on your own computer, so updating never touches them. Pull the latest image, replace the container, and rerun:

docker stop unlv-cpp-ide && docker rm unlv-cpp-ide
docker pull seancnc/unlv-cpp-ide && docker image prune -f
docker run -it --name unlv-cpp-ide -p 127.0.0.1:8135:8080 -v "$HOME/UNLV/cpp-workspace:/home/coder/workspace" seancnc/unlv-cpp-ide
